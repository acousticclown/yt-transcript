# Contributing to YT-Transcript

Thank you for your interest in contributing! This project is built with intention, day-by-day.

## Philosophy

Before contributing, please read [docs/vision.md](./docs/vision.md) to understand the project's core principles:
- Incremental development
- Free tools first
- Replaceable components
- Playful but focused UX
- Personal-first, OSS-friendly

## Getting Started

1. Fork the repository
2. Clone your fork
3. Set up the development environment (see [README.md](./README.md))
4. Create a branch for your contribution

## Development

### Backend
```bash
cd apps/api
npm install
npm run dev
```

### Frontend
```bash
cd apps/web
npm install
npm run dev
```

## Code Style

- Use TypeScript
- Follow existing code patterns
- Keep prompts in `packages/prompts/` (versioned as code)
- Write clear, readable code
- Add comments when logic is non-obvious

## Pull Requests

1. Keep PRs focused on one feature/fix
2. Update documentation if needed
3. Test your changes
4. Write clear commit messages

## Questions?

Open an issue for discussion. We're friendly and open to ideas!
